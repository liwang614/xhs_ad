# AI 工作指南

## 目标

在这个仓库内，优先保证“按用户当次需求做最小改动”，避免无关重构。

## 工作原则

1. 先确认改动边界，只改用户明确要求的部分。
2. 涉及数据库字段时，先核对真实表名和当前连接库。
3. 不把密钥写进 `config.json` 或代码，统一走环境变量。
4. 失败要可追踪：写清楚日志位置、记录最小必要错误信息。
5. 做完后给可执行验证步骤（SQL + 命令）。

## 与 AI 判定相关的约定

1. 目标表：`xhs_note`
2. 判定字段：`is_help_post`（`1=求助帖`，`0=非求助帖`，`NULL=未判定`）
3. 模型输出必须最终解析为 JSON，核心字段只认：
   - `{"is_help_post": 0}`
   - `{"is_help_post": 1}`
4. 判定失败时，不覆盖数据库，保持 `is_help_post=NULL`，并写错误日志。
